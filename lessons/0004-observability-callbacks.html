LiteLLM Proxy 网关 · 第 04 课

可观测性:用 callbacks 看见每次请求的花费与延迟

这一课服务于你的使命:生产网关跑起来后,你必须能回答"今天花了多少钱?哪个模型慢?哪把 key 在烧额度?"。 今天开 litellm_settings.callbacks 抽屉,把每次请求的花费、延迟、token、成败 自动吐到监控系统——出问题能查,花钱能算。

一、心智模型:callbacks 是"请求的旁路录像"

网关处理每个请求时,会把一份标准化记录(模型、token、花费、延迟、时间戳、元数据) 交给你登记的 callbacks,由它们转发到监控后端。你不改业务,只是在旁边"装了摄像头"。[1]

三个登记位都在 litellm_settings 里:

  • success_callback — 成功的请求往哪送。
  • failure_callback — 失败的请求往哪送。
  • callbacks — 成功+失败都送(很多集成用这个统一写法)。

二、关键洞察:两层可观测性,别只装一个

两层缺一不可 • 聚合指标(Prometheus) —— 趋势与告警。"过去 1 小时 QPS、错误率、各模型 p95 延迟、各 key 累计花费"。 用来盯大盘、设报警。[2]
• 单请求追踪(Langfuse) —— 逐条排障。点开某一次调用看它的 prompt、用了哪个部署、花了多少、慢在哪。 用来查具体那次为什么贵/为什么慢。[1]
口诀:Prometheus 看森林,Langfuse 看树木。

三、动手 A:Prometheus 大盘

LiteLLM 内置 Prometheus 导出器,开启后网关自带一个 GET /metrics 端点供抓取:[2]

config.yaml

litellm_settings:
  callbacks: ["prometheus"]
# 重启后直接看指标(无需额外服务即可验证端点)
curl http://0.0.0.0:4000/metrics | head -40

你会看到按 model / key / team 拆分的 spend、请求数、token、延迟、各部署剩余配额等。 生产里让 Prometheus 周期抓这个端点,再用 Grafana 出大盘 + 告警。

四、动手 B:Langfuse 逐条追踪

config.yaml

litellm_settings:
  callbacks: ["prometheus", "langfuse"]   # 两层一起开
export LANGFUSE_PUBLIC_KEY="pk-..."
export LANGFUSE_SECRET_KEY="sk-..."
export LANGFUSE_HOST="https://cloud.langfuse.com"   # 自托管则填你的地址
litellm --config config.yaml

随便调几次网关,去 Langfuse 控制台就能看到每一条调用的花费、延迟、用到的部署与完整对话。

生产隐私:记花费但不记内容

很多场景不能把用户 prompt 落到第三方。LiteLLM 可以关掉消息内容但保留花费/延迟统计:[1] 

litellm_settings:
  callbacks: ["langfuse"]
  turn_off_message_logging: true     # 脱敏:只记元数据与花费,不记 prompt/回复
