config.yaml 结构速查

压缩版,供随时回看。详细字段见官方 Config Settings。 口诀:模型 → 全局 → 网关 → 路由。

四个顶层抽屉

段 管什么 典型字段
model_list 有哪些模型、怎么连 model_name, litellm_params
litellm_settings 全局行为 cache, callbacks, request_timeout
general_settings 网关自身 master_key, database_url
router_settings 路由/可靠性 routing_strategy, fallbacks, num_retries

model_list 一项的解剖

- model_name: gpt-4o              # ① 对外名(客户端 "model" 写这个)
  litellm_params:
    model: openai/gpt-4o          # ② 真实串:提供商/模型
    api_key: os.environ/OPENAI_API_KEY   # ③ 从环境变量读,别写死
    api_base: https://...         # ④ 自定义 endpoint(国内/兼容模型必填)
    rpm: 100                      # ⑤ 可选:每分钟请求上限
    tpm: 100000                   # ⑤ 可选:每分钟 token 上限

常用 provider 前缀(model 字段)

openai/gpt-4o                 # OpenAI
anthropic/claude-3-5-sonnet   # Anthropic
openai/deepseek-chat          # OpenAI 兼容(配 api_base 指向厂商)
gemini/gemini-1.5-pro         # Google

国内模型基本走 openai/<模型> + api_base 这条兼容通法; 个别厂商有专属前缀,逐个实测后补到这里。

启动 & 调用

# 安装
pip install 'litellm[proxy]'

# 启动(默认 0.0.0.0:4000)
litellm --config config.yaml

# 调用(OpenAI 兼容)
curl http://0.0.0.0:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"hi"}]}'
